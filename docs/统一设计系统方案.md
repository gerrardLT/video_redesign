# 统一设计系统方案（Unified Design System）

> 目标：解决当前「一个 App 两套互不相干视觉身份」的精神分裂问题，建立**一套品牌语义 token + 两个表层皮肤**，让本地生活营销（商家端·浅色）与视频重塑（创作端·深色）同源、可互相跳转而不产生视觉断崖。
>
> 评审依据：huashu-design 5 维度评审。本方案遵循「好设计从已有上下文长出来」——不推翻现有投资（`--cine-*` 暖金体系、商家端暖橙移动体验），而是把两者收敛到同一套语义层。
>
> 状态：方案稿（待评审 → 分阶段落地）。所有 token 值均给出具体 hex / oklch，可直接粘贴。

---

## 0. 设计决策（先定方向，统领全文）

### 0.1 谁是主框架

依据 `merchant-billing-unification` 的既定结论与 `merchant/layout.tsx` 注释：

- **本地生活营销平台 = 主框架**（用户日常入口、商家工作台）
- **视频重塑 = 主框架下的「创作能力模块」**（/dashboard）

因此品牌基调应以「**本地生活商家**」为锚，视频重塑作为其下的「专业创作模式」存在，用**深色皮肤**表达「进入沉浸式创作」，而非另一个独立产品。

### 0.2 一个品牌，两个表层皮肤（已选定 Starbucks 方向）

商家皮肤的视觉方向经评审选定 **Starbucks「暖奶油 · 大地绿 · 会员金」**（依据 `docs/design-md/starbucks/DESIGN.md` 真实 DNA，原型见 `design-demos/locallife-starbucks.html`）。

| 皮肤 | 适用 | 默认底色 | 行动主色 | 仪式/高级色 |
|------|------|---------|---------|------------|
| **Light（商家皮肤）** | `/merchant/**`、`/`(落地)、`/login`、营销页 | 暖奶油 `#f2f0eb` | 大地绿 `#00754A`（CTA/选中）| 会员金 `#cba258`（会员/积分专用）|
| **Studio（创作皮肤）** | `/dashboard/**`（视频重塑模块） | 近黑电影底 `#0B0C0E` | 暖金 `#C7A877`（创作 accent）| — |

**金色是跨产品统一的那根线**：在商家皮肤里它是「会员/积分」的仪式色，在创作皮肤里它是「创作 accent」主色——同一族暖金（hue ≈ 38–45°），让用户从 `/merchant` 跳到 `/dashboard` 时感到「同一个品牌的两种气质」，而非两个产品。

两个皮肤**引用同一套语义 token 名**，只是取值不同（商家=浅底绿主、创作=深底金主），组件零改动即可换肤。

> 设计取舍：商家端的「行动色」用绿（信任、食欲、餐饮零售调性，且绿底白字对比度天然达标），金只保留给会员仪式——这正是 Starbucks 体系「绿做主、金做 Rewards 仪式」的纪律。绿是商家浅皮肤专属，不进入创作端。

### 0.3 统一字体与图标策略

- **字体**：正文/UI 用 `Noto Sans SC`；展示/营销标题用 `Noto Serif SC`（编辑感）；等宽 `Geist Mono`。**全部通过 `next/font` 真正加载**（当前落地页引用了 Noto 但从未加载，必须修复）。
- **图标**：功能控件**统一用 `lucide-react`**（线性）。Emoji **仅允许作为「内容/行业标识」**（如行业选择 🍲🧋☕），**禁止用作功能按钮/导航/标题装饰图标**。

## 1. 颜色系统

### 1.1 品牌原色（两族 + 一条金线）

商家皮肤的行动色用**大地绿族**；金色作为**跨产品共享线**（商家=会员，创作=创作 accent）单独成族。

**绿族（商家行动/信任 · Starbucks 四级绿）**

| Token | Hex | 用途 |
|-------|-----|------|
| `--green-accent` | `#00754A` | **商家行动主色**（CTA、选中态；白字 AA ≈ 4.8:1）|
| `--green-sb`     | `#006241` | 标题、品牌强调 |
| `--green-house`  | `#1E3932` | 深绿 band（会员头部、特性区背景）|
| `--green-active` | `#005C3A` | 按下态 |
| `--green-light`  | `#D4E9E2` | 浅绿底（选中底、成功 dim、标签）|

**金线（跨产品共享 · 暖金）**

| Token | Hex | 用途 |
|-------|-----|------|
| `--gold`         | `#CBA258` | 商家=会员/积分仪式；创作端别名 `--brand-gold` |
| `--gold-light`   | `#DFC49D` | 金 hover / 边框 |
| `--gold-lightest`| `#FAF6EE` | 金标签底、会员区浅底 |
| `--brand-gold`   | `#C7A877` | **创作皮肤主 accent**（沿用现有 `--cine-gold`，配 ink 字）|
| `--brand-gold-2` | `#B89765` | 创作皮肤 hover |
| `--ink-on-gold`  | `#1A1408` | 金底之上的深色字（创作端按钮文字）|

> 说明：`--gold`(#CBA258) 与 `--brand-gold`(#C7A877) 同属暖金族，前者略亮用于浅皮肤会员仪式，后者略沉用于深皮肤创作 accent——肉眼一眼识别为同源，这就是「跨产品那根线」。绿只服务商家浅皮肤，不进创作端。

### 1.2 中性阶（Neutral · 双皮肤各一套）

浅皮肤用 Starbucks 暖奶油画布（非冷白，承载食欲/温度）；深皮肤沿用电影质感近黑冷青。

```
/* 浅皮肤中性（暖奶油，Starbucks 调） */
--canvas:#F2F0EB   /* 主画布·暖奶油 */   --ceramic:#EDEBE9  /* 区块分隔/次级底 */
--n-0:#FFFFFF      /* 卡片/模态白 */      --neutral-cool:#F9F9F9 /* 下拉/表单浅底 */
--hair:#E7E7E7     /* 1px 分隔线 */        --border-strong:#D6DBDE
--ink:rgba(0,0,0,.87)   /* 主文字·非纯黑 */  --ink-soft:rgba(0,0,0,.58) /* 次级文字 */
--ink-faint:rgba(0,0,0,.40)               /* 三级/占位 */

/* 深皮肤中性（电影近黑，沿用 --cine-*） */
--d-bg:#0B0C0E  --d-bg-soft:#0F1113  --d-surface:#141619
--d-line:rgba(255,255,255,.055)  --d-line-2:rgba(255,255,255,.09)
--d-text:#E9E6DF  --d-text-2:#8C8B85  --d-text-3:#5A5B57
```

> 关键：浅皮肤画布是**暖奶油 `#f2f0eb` 而非纯白**——这是 Starbucks 体系的「载重色」，奶油底让餐饮内容更有食欲与温度，白卡片浮在奶油底上形成层级（靠表面差异，不靠重阴影）。

### 1.3 语义状态色（双皮肤统一语义，取值各自适配）

| 语义 | 浅皮肤（Starbucks 调） | 深皮肤（沿用 cine） |
|------|--------|--------------------|
| success/完成 | `#00754A`(=绿主) / 底 `#D4E9E2` | `--cine-green #7E9B86` / 底 dim |
| warning/进行中·排队 | `#B5791F` / 底 `#FDF3E3` | `--cine-amber #C2965A` / 底 dim |
| danger/失败 | `#C82014` / 底 `#FBEAE5` | `--cine-red #C26B58` / 底 dim |
| info | `#2F6F9E` / 底 `#E6F0F7` | `#6FA8C7` / 底 dim |

> 替换当前 `dashboard/page.tsx` 里 7 色彩虹状态徽章（yellow/blue/orange/emerald/purple/red）→ 收敛到以上 4 语义。注意：完成态用绿主色，与「会员金」区分清楚——金永远只代表会员/积分，不代表「成功」。

### 1.4 语义层（Semantic Tokens · 组件只引用这一层）

组件**只用语义名**，不直接写品牌/中性原色。两皮肤通过覆盖语义名取值实现换肤。

```css
/* ===== 浅皮肤（默认，商家 + 落地 + 登录）· Starbucks 暖奶油 ===== */
:root {
  --bg:            var(--canvas);     /* 暖奶油 #f2f0eb */
  --surface:       var(--n-0);        /* 白卡片 */
  --surface-2:     var(--ceramic);    /* 次级底 #edebe9 */
  --border:        var(--hair);
  --border-strong: var(--border-strong);
  --text:          var(--ink);
  --text-2:        var(--ink-soft);
  --text-3:        var(--ink-faint);

  --primary:        var(--green-accent);  /* 行动主色=大地绿 #00754A */
  --primary-hover:  var(--green-sb);      /* #006241 */
  --primary-active: var(--green-active);  /* #005C3A */
  --on-primary:     #FFFFFF;              /* 绿底白字（AA ≈4.8:1） */

  --accent:         var(--green-accent);  /* 选中态/强调=绿 */
  --accent-dim:     var(--green-light);   /* 浅绿底 */

  /* 会员仪式专用金（仅会员/积分语境用） */
  --membership:        var(--gold);
  --membership-dim:    var(--gold-lightest);
  --membership-strong: #8A6D2F;           /* 金底之上的深字 */

  --success:#00754A; --success-dim:var(--green-light);
  --warning:#B5791F; --warning-dim:#FDF3E3;
  --danger:#C82014;  --danger-dim:#FBEAE5;
  --info:#2F6F9E;    --info-dim:#E6F0F7;

  --radius: 14px;          /* 卡片 12-14；按钮走 --radius-pill 全胶囊 */
}

/* ===== 创作皮肤（视频重塑 /dashboard 根节点挂 data-surface="studio"） ===== */
[data-surface="studio"] {
  --bg:            var(--d-bg);
  --surface:       var(--d-surface);
  --surface-2:     var(--d-bg-soft);
  --border:        var(--d-line);
  --border-strong: var(--d-line-2);
  --text:          var(--d-text);
  --text-2:        var(--d-text-2);
  --text-3:        var(--d-text-3);

  --primary:        var(--brand-gold);   /* 创作皮肤主色=暖金 */
  --primary-hover:  var(--brand-gold-2);
  --primary-active: var(--brand-gold-2);
  --on-primary:     var(--ink-on-gold);  /* 金底配深 ink 字（高对比，统一！） */

  --accent:         var(--brand-gold);
  --accent-dim:     rgba(199,168,119,.14);

  /* 创作端金即主色，会员语义复用主金 */
  --membership:        var(--brand-gold);
  --membership-dim:    rgba(199,168,119,.14);
  --membership-strong: var(--brand-gold);

  --success:#7E9B86; --success-dim:rgba(126,155,134,.13);
  --warning:#C2965A; --warning-dim:rgba(194,150,90,.14);
  --danger:#C26B58;  --danger-dim:rgba(194,107,88,.13);
  --info:#6FA8C7;    --info-dim:rgba(111,168,199,.13);

  --radius: 12px;
}
```

> 关键统一点：① 两皮肤主按钮都用 `--on-primary`——浅皮肤绿底白字、创作皮肤金底 ink 字，**各自都满足对比度**，一并修掉了「登录页金底白字 1.9:1」和「落地 ink 字/后台白字自相矛盾」两个问题。② **金 = 会员/高级**这一语义在两端一致（`--membership`），是跨产品识别线。

## 2. 字体系统（Typography）

### 2.1 字族（必须真正加载）

| Token | 字体 | 用途 |
|-------|------|------|
| `--font-sans` | `Noto Sans SC`, -apple-system, system-ui, sans-serif | 正文、UI、按钮、表单 |
| `--font-display` | `Noto Serif SC`, serif | 营销/展示标题（落地页 hero、章节大标题）|
| `--font-mono` | `Geist Mono`, ui-monospace, monospace | 代码、数字、技术信息 |

> ⚠️ 现状 bug：落地页 CSS 写了 `Noto Serif SC`/`Noto Sans SC`，但 `layout.tsx` 只加载了 Geist（且仅 latin 子集，不含中文）。结果 hero 大标题实际回退到系统宋体，电影质感的视觉锚点失效。**Phase 0 必修**。

加载方式（`layout.tsx`，用 `next/font/google`）：

```tsx
import { Noto_Sans_SC, Noto_Serif_SC, Geist_Mono } from 'next/font/google'

const sans = Noto_Sans_SC({ subsets: ['latin'], weight: ['400','500','700'], variable: '--font-sans', display: 'swap' })
const serif = Noto_Serif_SC({ subsets: ['latin'], weight: ['500','700'], variable: '--font-display', display: 'swap' })
const mono = Geist_Mono({ subsets: ['latin'], variable: '--font-mono' })
// <html className={`${sans.variable} ${serif.variable} ${mono.variable}`}>
```

> 中文字形较大，注意按需 weight；若首屏体积敏感，可改自托管 woff2 + `font-display: swap` + 子集化（pyftsubset / fontmin）。

### 2.2 字阶（Type Scale · 1.25 模数）

| Token | size / line-height | 用途 |
|-------|--------------------|------|
| `--text-xs`  | 12 / 16 | 辅助、徽章、时间戳 |
| `--text-sm`  | 14 / 20 | 正文次级、表单标签 |
| `--text-base`| 16 / 24 | 正文基准（移动端表单建议 ≥16 防 iOS 缩放）|
| `--text-lg`  | 18 / 26 | 小标题 |
| `--text-xl`  | 20 / 28 | 卡片标题 |
| `--text-2xl` | 24 / 32 | 页面标题 |
| `--text-3xl` | 30 / 38 | 区块标题 |
| `--text-4xl` | 40 / 48 | 营销中标题（display）|
| `--text-5xl` | 56 / 60 | 落地 hero（display, `clamp(36px,5vw,56px)`）|

规则：标题与正文对比 ≥2.5×；display 字号必须配 `--font-display`；正文 `text-wrap: pretty`，标题 `text-balance`。

## 3. 间距 / 圆角 / 阴影 / 动效

### 3.1 间距（8pt 网格）

`--space-1:4 · 2:8 · 3:12 · 4:16 · 5:24 · 6:32 · 7:48 · 8:64 · 9:96`（px）。间距只取这些档，禁止随手 13/18/22。

### 3.2 圆角（Radius）

```
--radius-sm: calc(var(--radius) - 6px)   /* 控件内小元素 */
--radius-md: calc(var(--radius) - 3px)   /* 输入框、按钮 */
--radius-lg: var(--radius)               /* 卡片：浅皮肤=14, 创作皮肤=12 */
--radius-xl: calc(var(--radius) + 6px)   /* 大卡片、模态 */
--radius-pill: 9999px                    /* 徽章、Tag、底部导航项 */
```

> 商家端「大圆角亲和」与创作端「克制」通过 `--radius` 基准值差异表达（14 vs 12），同一套档位名。

### 3.3 阴影（Elevation）

```
/* 浅皮肤：柔和暖阴影 */
--shadow-sm:0 1px 2px rgba(31,28,23,.06)
--shadow-md:0 4px 16px rgba(31,28,23,.08)
--shadow-lg:0 12px 40px rgba(31,28,23,.12)
/* 创作皮肤：深色重阴影（沿用现有 lp 卡片手感） */
--shadow-sm-d:0 2px 8px rgba(0,0,0,.3)
--shadow-md-d:0 12px 40px rgba(0,0,0,.4)
--shadow-lg-d:0 30px 80px rgba(0,0,0,.5)
```

### 3.4 动效（Motion · 复用落地页已有的好曲线）

```
--ease-out: cubic-bezier(.16,1,.3,1)      /* 入场、揭示 */
--ease-spring: cubic-bezier(.23,1.4,.32,1)/* 弹性强调 */
--dur-fast:150ms  --dur-base:300ms  --dur-slow:600ms  --dur-reveal:900ms
```

规则：优先 `transform`/`opacity`；一次编排的页面入场 > 散落微交互；尊重 `prefers-reduced-motion`（关闭非必要动画）。

## 5. 组件与 shadcn 绑定

shadcn/ui 通过 CSS 变量驱动。把 shadcn 的 token **指向本系统语义层**，两皮肤切换即自动生效，无需改组件：

```css
@theme inline {
  --color-background: var(--bg);
  --color-foreground: var(--text);
  --color-card: var(--surface);
  --color-card-foreground: var(--text);
  --color-primary: var(--primary);
  --color-primary-foreground: var(--on-primary);
  --color-secondary: var(--surface-2);
  --color-muted: var(--surface-2);
  --color-muted-foreground: var(--text-2);
  --color-accent: var(--accent-dim);
  --color-accent-foreground: var(--text);
  --color-destructive: var(--danger);
  --color-border: var(--border);
  --color-input: var(--border);
  --color-ring: var(--primary);
  --font-sans: var(--font-sans);
  --font-mono: var(--font-mono);
}
```

> 这样 admin 后台与商家端大量使用的 shadcn `Button/Card/Input/Badge/Dialog` 自动继承品牌色，不再是「未改的 shadcn 默认近黑」。

**核心组件规范（摘要）**：
- Button：primary(`--primary`+`--on-primary`) / secondary(`--surface-2`) / ghost / outline(`--border`) / danger(`--danger`)；高 40/`h-10`，移动端 48/`h-12`；圆角 `--radius-md`。
- Input：`--surface` 底 + `--border`，focus `ring-1 --primary`；移动端 `text-base` 防缩放。
- Card：`--surface` + `--border` + `--shadow-sm`，圆角 `--radius-lg`。
- Badge/状态：用 §1.3 语义色，文字色取对应深色，底取 `*-dim`。
- 空/载/错三态：统一插画/skeleton 风格（skeleton 用 `--surface-2` 脉冲）。
