# AI 视频重塑工具 MVP 产品需求文档

## 1. 产品概述

### 1.1 产品名称

AI 视频重塑工具 MVP

### 1.2 产品定位

本产品是一款面向短视频创作者的 AI 视频重塑工具。用户上传一条已有视频后，系统自动解析视频内容，生成结构化分镜脚本。用户可以编辑每个分镜的人物、场景、对白和生成提示词，并使用 AI 生成新的分镜视频，最后将所有分镜片段合并导出为完整视频。

### 1.3 MVP 目标

MVP 阶段的目标不是做完整平台，而是验证核心流程是否成立：

用户是否愿意上传视频；
系统是否能自动生成可编辑的分镜脚本；
用户是否能通过编辑 prompt 和参考素材生成满意的视频片段；
Seedance 2.0 生成链路是否稳定；
完整视频合并导出是否可用；
单位成本是否可控。

### 1.4 MVP 核心闭环

上传原视频
→ 系统自动解析分镜脚本
→ 用户编辑分镜内容
→ 用户上传参考素材 / 生成人物图
→ 用户生成单个或多个分镜视频
→ 系统合并导出完整视频
→ 用户下载成片

## 2. 用户角色

### 2.1 普通用户

短视频创作者、运营人员、电商内容人员、广告素材制作人员。

主要诉求：

快速把已有视频改写成新风格、新画面或新版本；
不想从零写脚本；
希望能基于原视频自动拆分分镜；
希望控制人物、场景和参考图；
希望最终导出完整视频。

### 2.2 管理员

平台运营人员或开发者。

主要诉求：

查看用户项目；
查看生成任务状态；
处理失败任务；
查看 API 成本；
手动调整用户额度；
排查生成失败原因。

## 4. 核心用户流程

### 4.1 创建项目流程

用户登录
→ 点击「新建项目」
→ 上传本地视频
→ 系统创建项目
→ 显示解析中状态
→ 解析完成后进入分镜列表页

### 4.2 分镜编辑流程

用户进入项目详情页
→ 查看系统生成的分镜列表
→ 展开某个分镜
→ 查看人物、场景、对白、音频、prompt
→ 修改分镜内容
→ 系统自动保存
→ 用户点击生成视频

### 4.3 素材使用流程

用户展开某个分镜
→ 上传参考图片
→ 系统加入素材库
→ 素材显示为 [图1]、[图2]
→ 用户点击素材缩略图
→ 系统在 prompt 光标位置插入 [图N]
→ 生成时系统自动关联对应图片

### 4.4 视频生成流程

用户点击「生成视频」
→ 系统检查 prompt、素材、参数
→ 检查积分是否足够
→ 创建生成任务
→ 任务进入队列
→ 调用 Seedance 2.0
→ 前端显示生成中
→ 生成完成后展示视频预览
→ 用户可下载或重试

### 4.5 批量生成流程

用户点击「开始批量生成」
→ 系统检查所有未完成分镜
→ 已成功生成的分镜自动跳过
→ 未生成分镜依次进入队列
→ 每个分镜独立生成
→ 失败分镜可单独重试
→ 所有分镜完成后允许合并导出

### 4.6 合并导出流程

用户点击「合并导出」
→ 系统检查所有分镜是否已有视频
→ 使用 FFmpeg 统一规格并合并
→ 生成完整视频
→ 用户预览完整视频
→ 用户下载 MP4 文件

## 8. 积分与扣费规则

### 8.1 MVP 扣费方式

按分镜视频生成扣费。

解析脚本免费；
上传素材免费；
编辑 prompt 免费；
合并导出免费；
生成视频扣费；
重新生成扣费；
系统失败返还。

### 8.2 扣费流程

用户点击生成
→ 系统估算所需积分
→ 检查余额
→ 冻结积分
→ 创建任务
→ 任务成功后正式扣除
→ 任务失败后返还冻结积分

### 8.3 失败返还规则

系统错误：返还；
API 超时：返还；
素材读取失败：返还；
内容安全拒绝：按平台规则处理；
用户主动取消排队：返还；
用户主动重生成：不返还上一条成功任务。

## 9. 任务状态机

生成任务状态：

created
→ queued
→ credit_reserved
→ submitted
→ generating
→ succeeded
→ failed
→ canceled
→ refunded

状态规则：

queued 状态可以取消；
submitted 和 generating 状态不能取消；
succeeded 状态不可重复扣费；
failed 状态可以重试；
canceled 状态需要返还积分；
refunded 状态不可再次退款。

---

## 10. 异常处理

### 10.1 上传失败

提示用户重新上传。

### 10.2 视频解析失败

允许用户重新解析。
如果再次失败，提示用户更换视频。

### 10.3 分镜脚本生成失败

展示基础分镜结构，允许用户手动填写 prompt。

### 10.4 素材上传失败

提示失败原因，包括格式不支持、文件过大、网络异常。

### 10.5 视频生成失败

展示失败原因。
支持重新生成。
如果是系统原因，自动返还积分。

### 10.6 合并导出失败

允许用户重新导出。
后台记录失败原因。

Sonner（Toast 通知）；
Lucide React（图标）。

### 11.2 后端

已使用：

Next.js API Route Handlers；
SQLite（via Prisma 7.8 + @prisma/adapter-libsql + @libsql/client）；
Redis 7（Docker）；
BullMQ 5.x（10 个队列、11 个 Worker）；
阿里云 OSS（ali-oss）；
FFmpeg（本地 execFile 调用）；
火山引擎方舟 Seedance 2.0 API（doubao-seedance-2-0-260128）；
通义千问 qwen3.7-plus（DashScope 视频直传分析）；
FLUX API（人物图生成）。

### 11.3 存储

原视频、normalized 视频、关键帧、生成视频、合并视频、音频切片都存阿里云 OSS。
数据库只保存 URL、状态和元数据。
AI 生成资产设置 14 天过期，由 asset-cleanup Worker 定时清理。

### 11.4 队列

视频解析、视频生成、人物图生成、合并导出、链接下载、人脸检测、虚拟人像入库全部走异步队列。
商业化功能（资产清理、订单过期、通知提醒）通过 Repeatable Jobs 定时触发。
不使用同步接口等待视频生成完成。

## 15. MVP 成功指标

### 产品指标

上传视频用户中，超过 50% 完成分镜解析；
完成分镜解析用户中，超过 40% 点击生成视频；
点击生成视频用户中，超过 60% 完成至少一个分镜生成；
完成分镜生成用户中，超过 40% 完成合并导出。

### 技术指标

分镜解析平均耗时小于 3 分钟；
单分镜任务状态可追踪率 100%；
重复扣费事故为 0；
生成失败后积分返还准确率 100%；
合并导出成功率大于 90%。

### 成本指标

每个项目可统计完整成本；
每个分镜可统计生成成本；
每个用户可统计积分消耗；
系统可识别失败重试成本。

## 16. 总结

MVP 阶段只做一个核心目标：

让用户上传一个视频，系统自动拆成分镜，用户简单编辑后，可以生成新的视频片段，并合并导出完整视频。

第一版不要追求功能大而全。
不做自动发布。
不做复杂抖音解析。
不做企业协作。
不做高级内容中心。
不做过度复杂的角色一致性系统。

MVP 的重点是验证：

视频解析是否可用；
分镜编辑是否顺畅；
Seedance 生成是否稳定；
成本是否可控；
用户是否愿意为生成结果付费。

只要这个闭环成立，后续再加一键分发、案例中心、智能助手、抖音链接和高级角色库。
