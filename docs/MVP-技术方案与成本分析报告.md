## 1. 技术栈方案

### 1.1 推荐技术架构

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| **前端框架** | Next.js 15 + React 19 | SSR/SSG 混合渲染，Vercel 原生支持，生态成熟 |
| **UI 框架** | Tailwind CSS + shadcn/ui | 开发效率高，组件库丰富，移动端适配方便 |
| **状态管理** | Zustand | 轻量、TypeScript 友好、无 boilerplate |
| **视频播放** | Video.js 或 Plyr | 开源、可定制、支持多格式 |
| **后端 API** | Next.js API Routes + Route Handlers | 与前端同项目，部署简单，Serverless 架构 |
| **数据库** | SQLite (via Prisma + libsql) | 零运维、零月费、单文件部署，未来可无缝迁移 Turso 云端 |
| **缓存/队列** | 自建 Redis + BullMQ | 本地部署，零月费，队列功能强大，延迟任务和重试 |
| **对象存储** | 阿里云 OSS | 视频/图片存储，CDN 加速分发 |
| **视频处理** | FFmpeg (Serverless 函数 / 独立服务) | 抽帧、转码、合并，开源免费 |
| **AI 视频生成** | Seedance 2.0 API (火山引擎方舟平台) | 官方 doubao-seedance-2-0-260128 模型，$0.08/秒(720p) |
| **AI 视频解析** | 通义千问 qwen3.7-plus (DashScope) | 视频直传模式，模型直接理解视频内容 |
| **AI 图像生成** | FLUX 1.1 Pro (via fal.ai) | 人物参考图生成，质量高成本低 |
| **部署平台** | Vercel (前端+API) + Railway/Fly.io (长任务) | Serverless + 持久化服务混合 |
| **监控** | Sentry + Vercel Analytics | 错误追踪 + 性能监控 |

## 6. 成本优化空间分析

### 6.1 视频生成成本优化（影响最大）

| 优化策略 | 节省比例 | 实施难度 | 说明 |
|----------|---------|---------|------|
| **🔥 等待官方 API 开放** | ~50% | 低 | 官方 $0.08 vs EvoLink $0.161，直接减半 |
| **🔥 使用 V2V 模式替代 T2V** | ~40% | 中 | 有参考视频时走 V2V（¥28/M vs ¥46/M tokens） |
| **使用 Fast 替代 Standard** | ~20% | 低 | Fast 720p $0.161 vs Standard $0.199 |
| **草稿用 480p，终稿用 720p** | 30-50% | 中 | 用户预览时用 480p，满意后再生成 720p |
| **智能分镜时长** | 10-20% | 中 | 短场景用 4-6s 而非统一 8s |
| **Prompt 预校验减少重试** | 15-20% | 中 | 提交前 AI 检查 prompt 质量，降低失败率 |
| **批量折扣/大客户协议** | 20-40% | 高 | 直接与 ByteDance 签订批量协议 |
| **缓存相似生成** | 5-10% | 高 | 相同 prompt+参数命中缓存时跳过生成 |
| **即梦月卡 (¥69) 做小批量** | 70-80% | 中 | 适合初期验证，非规模化方案 |

**最佳实践组合（官方 API 开放后）：V2V 模式 + 720p + 智能时长 + 预校验 → 综合节省 50-65%**

优化后单项目成本（官方 API）：$7.68 → **$3.5-5.0（约 ¥25-36）**
优化后单项目成本（第三方）：$15.69 → **$8.6-10.2（约 ¥62-74）**

### 6.2 视频解析成本优化

| 优化策略 | 节省 | 说明 |
|----------|------|------|
| 视频直传模式替代帧图片模式（已采用） | 60-80% | 减少 token 输入量，模型直接理解视频 |
| 通义千问 qwen3.7-plus 替代 GPT-4o | 80-90% | 国产模型成本远低于 GPT-4o |
| 使用 prompt cache | 30-50% | 系统提示词缓存，减少重复输入成本 |

### 6.3 图像生成成本优化

| 优化策略 | 节省 | 说明 |
|----------|------|------|
| FLUX Schnell 替代 FLUX Pro | 50-70% | $0.003 vs $0.06 每张（质量略降） |
| 只在用户请求时生成 | 变动 | 不自动生成，按需生成 |
| 本地缓存人物图 | 20% | 同项目复用不重复生成 |

### 6.4 基础设施优化

## 10. 关键结论

1. **成本结构极度集中**：Seedance 2.0 视频生成占总成本 96-97%，其他环节成本可忽略不计
2. **火山引擎官方定价是基准**：720p T2V ~$0.08/秒（¥28/M tokens），1080p ~$0.14/秒（¥46/M tokens）
3. **V2V 模式比 T2V 便宜 40%**：本产品「基于原视频重塑」天然适合 V2V 模式，是重要的成本优势
4. **当前第三方有 100-280% 溢价**：EvoLink Fast $0.161/秒（+100%），fal.ai $0.2419/秒（+200%）
5. **单项目成本双档位**：官方 API ≈ ¥57（T2V）/ ¥33（V2V）；第三方 ≈ ¥113（EvoLink Fast）
6. **15 秒短视频成本约 ¥6-18**（取决于渠道和模式）
7. **基础设施月费 $10-35**（MVP 阶段，SQLite 零成本 + Redis Docker），不是成本瓶颈
8. **官方 API 开放是最大降本杠杆**：开放后成本直接减半，建议密切关注火山引擎动态
9. **建议售价 ¥99-129/项目（2 分钟视频）**，对应 44-57% 毛利率
