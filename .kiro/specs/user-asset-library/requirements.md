# Requirements Document

## Introduction

用户资产库功能：提供一个用户级别的资产管理系统，将视频解析过程中生成的人物形象图（Seedream 虚拟角色参考图）自动存入用户资产库，支持多种资产分类（角色图、素材、音频等），并实现跨项目复用。用户可以在新项目中直接选择已有资产而无需重新生成，同时支持查看、分类筛选、搜索和删除等管理操作。

## Glossary

- **Asset_Library**: 用户资产库，归属于单个用户的全部可复用资产集合，跨项目共享
- **Asset**: 资产条目，包含文件 URL、分类类型、缩略图、元数据等信息的单个可复用资源
- **Asset_Type**: 资产分类枚举，包括 CHARACTER（角色图）、MATERIAL（素材）、AUDIO（音频）
- **Character_Image**: 角色图资产，由 Seedream 模型生成的虚拟角色参考图，用于视频生成时的人物一致性参考
- **Asset_Library_Service**: 资产库服务端模块，负责资产的创建、查询、分类、搜索和删除
- **Asset_Library_UI**: 资产库前端界面，提供资产浏览、筛选、搜索和管理交互
- **OSS_Storage**: 阿里云 OSS 对象存储服务，存放资产文件的持久化存储层
- **Project_Character_Picker**: 项目人物选择器，用户在新项目中从资产库选取已有角色图复用的交互组件

## Requirements

### Requirement 1: 自动入库角色图

**User Story:** 作为用户，我希望视频解析生成的人物形象图自动存入我的资产库，以便后续在其他项目中复用而无需重新生成。

#### Acceptance Criteria

1. WHEN a Character_Image is generated by the Seedream model during video parsing, THE Asset_Library_Service SHALL create an Asset record with type CHARACTER and associate it with the owning user
2. WHEN a Character_Image Asset is auto-created, THE Asset_Library_Service SHALL store the Character name, the OSS HTTPS URL, and generate a thumbnail URL in the Asset record
3. WHEN a user manually regenerates a Character_Image via the character panel, THE Asset_Library_Service SHALL update the existing CHARACTER Asset or create a new one if none exists
4. IF the Seedream image generation fails, THEN THE Asset_Library_Service SHALL NOT create an Asset record and SHALL log the failure reason

### Requirement 2: 资产分类管理

**User Story:** 作为用户，我希望资产库中的资产按类型分类，以便快速定位不同种类的资产。

#### Acceptance Criteria

1. THE Asset_Library_Service SHALL support the following Asset_Type categories: CHARACTER（角色图）, MATERIAL（素材）, AUDIO（音频）
2. WHEN an Asset is created, THE Asset_Library_Service SHALL assign exactly one Asset_Type to the Asset record
3. THE Asset_Library_UI SHALL display Asset_Type filter tabs allowing the user to view assets filtered by a single category
4. WHEN a user selects an Asset_Type filter, THE Asset_Library_UI SHALL display only assets matching the selected category

### Requirement 3: 跨项目复用角色图

**User Story:** 作为用户，我希望在新项目中能选择已有的角色图作为人物参考图，以便保持角色一致性而无需重新生成。

#### Acceptance Criteria

1. WHEN a user creates or edits a Character in a project, THE Project_Character_Picker SHALL display available CHARACTER type assets from the user's Asset_Library
2. WHEN a user selects a CHARACTER Asset from the Asset_Library for a project Character, THE Asset_Library_Service SHALL copy the Asset OSS URL to the Character.imageUrl field
3. WHEN a CHARACTER Asset is reused in a project, THE Asset_Library_Service SHALL NOT duplicate the file in OSS_Storage; the same HTTPS URL SHALL be referenced
4. THE Project_Character_Picker SHALL display the asset name, thumbnail, and creation date for each available CHARACTER Asset

### Requirement 4: 资产列表查看

**User Story:** 作为用户，我希望能浏览我所有的资产，以便了解可复用资源的全貌。

#### Acceptance Criteria

1. THE Asset_Library_UI SHALL display a paginated list of all assets belonging to the authenticated user
2. THE Asset_Library_UI SHALL display each Asset's thumbnail, name, type, associated project name, and creation date
3. WHEN the asset list is loaded, THE Asset_Library_Service SHALL return assets sorted by creation date in descending order (newest first)
4. THE Asset_Library_Service SHALL support pagination with a configurable page size (default 20 items per page)

### Requirement 5: 资产搜索

**User Story:** 作为用户，我希望能通过关键字搜索资产，以便快速找到目标资产。

#### Acceptance Criteria

1. WHEN a user enters a search keyword, THE Asset_Library_Service SHALL return assets whose name contains the keyword (case-insensitive partial match)
2. WHEN a search keyword is combined with an Asset_Type filter, THE Asset_Library_Service SHALL return only assets matching both the keyword and the selected type
3. WHEN the search keyword is empty, THE Asset_Library_Service SHALL return the full unfiltered asset list (respecting other active filters)
4. THE Asset_Library_UI SHALL provide a text input for keyword search with debounced input (300ms delay before triggering search)

### Requirement 6: 资产删除

**User Story:** 作为用户，我希望能删除不再需要的资产，以便保持资产库整洁。

#### Acceptance Criteria

1. WHEN a user requests to delete an Asset, THE Asset_Library_UI SHALL display a confirmation dialog before proceeding
2. WHEN deletion is confirmed, THE Asset_Library_Service SHALL remove the Asset record from the database
3. WHEN a CHARACTER Asset is deleted, THE Asset_Library_Service SHALL NOT remove the corresponding file from OSS_Storage if any project Character still references the same URL
4. IF a user attempts to delete an Asset that does not belong to the authenticated user, THEN THE Asset_Library_Service SHALL return a 403 Forbidden error
5. WHEN an Asset is successfully deleted, THE Asset_Library_UI SHALL remove the item from the displayed list without requiring a full page reload

### Requirement 7: 资产库数据模型扩展

**User Story:** 作为开发者，我希望现有 Asset 数据模型能支持用户级别的跨项目资产管理，以便在不破坏现有功能的前提下实现资产库。

#### Acceptance Criteria

1. THE Asset_Library_Service SHALL store assets with a required userId field to establish user-level ownership independent of project scope
2. THE Asset_Library_Service SHALL store an optional projectId field to track which project originally generated the asset
3. THE Asset_Library_Service SHALL store a displayName field derived from the Character name or file name for search and display purposes
4. THE Asset_Library_Service SHALL maintain backward compatibility with existing project-scoped Asset records by preserving all existing fields and relationships
5. WHEN querying the Asset_Library, THE Asset_Library_Service SHALL filter by userId to ensure users only access their own assets

### Requirement 8: 资产库入口与导航

**User Story:** 作为用户，我希望能从主界面方便地进入资产库页面，以便随时管理我的可复用资产。

#### Acceptance Criteria

1. THE Asset_Library_UI SHALL be accessible from the dashboard sidebar navigation as a top-level menu item labeled "资产库"
2. WHEN the user navigates to the Asset_Library page, THE Asset_Library_UI SHALL load and display the user's assets with default sorting and no active filters
3. THE Asset_Library_UI SHALL display the total count of assets alongside each Asset_Type filter tab
